Admin

2FA required

Полный доступ

  • /admin Dashboard: users, businesses, pending queue
  • /admin/users Список, поиск, change role, ban
  • /admin/businesses Approve, reject, hide, view public page
  • /admin/introductions Approve, reject, add admin note
  • /admin/cards List, revoke
  • /admin/categories CRUD категорий
  • /admin/audit Read-only audit log

Особенности

  • Требуется 2FA Явно указано для admin panel MVP
  • Управляет статусами ACTIVE, BLOCKED, UNDER REVIEW, PUBLISHED, HIDDEN и др.

Логика access control

Основная модель доступа в MVP строится вокруг пяти состояний: Guest, Free Member, VIP Member, Business и Admin. При этом каталог и verify-card частично публичные, а ценность членства раскрывается через доступ к offers, digital card, VIP-функциям и Business Introductions.

Что важно для реализации

Для фронтенда удобно держать матрицу доступа отдельно: route → allowedRoles → fallback UI. Для gated sections внутри одной страницы лучше использовать не redirect, а условный рендер блоков: teaser, upgrade prompt, sign-in prompt, admin actions.

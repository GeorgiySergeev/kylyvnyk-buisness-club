# I18N

MVP launches with three locales: `en`, `ru`, and `uk`.

## Routing

- All user-facing routes use `/{locale}/...`.
- `/` redirects to `/en`.
- Slugs stay shared across locales for MVP, for example `/en/directory`,
  `/ru/directory`, and `/uk/directory`.
- `SUPPORTED_LOCALES` in `src/components/layout/navigation.ts` is the runtime
  source of truth for enabled locales.

## Messages

- Message files live in `messages/<locale>/<namespace>.json`.
- `en` is the source namespace shape.
- `ru` and `uk` must contain the same key set as `en`.
- `pnpm test` runs `tests/i18n/message-parity.test.ts` to enforce key parity.

## Copy Review

Russian and Ukrainian launch copy may be committed as draft implementation
copy, but legal-sensitive public text must be reviewed by a human before
production release.
